GitHub 블로그를 운영하다 보면 글마다 이미지를 따로 관리하고 싶어집니다. 하지만 GitHub 웹사이트를 아무리 뒤져봐도 '새 폴더 만들기' 버튼이 보이지 않아 당황하셨을 텐데요.

오늘은 GitHub 웹 인터페이스에서 **파일 만들기 기능을 이용해 감쪽같이 새 폴더를 만드는 꼼수**를 단계별로 정리해 보겠습니다.

---

## 1단계: 시작 위치로 이동하기

우선 폴더를 만들고자 하는 위치로 이동해야 합니다. 이미지 관리를 위한 것이라면 보통 다음 경로로 들어갑니다.

> **경로:** `assets` / `img` / `posts` /

---

## 2단계: 'Create new file' 선택하기

폴더 전용 버튼이 없으므로 '파일 만들기' 메뉴를 빌려와야 합니다.

1. 오른쪽 상단의 **Add file** 버튼을 클릭합니다.
2. 메뉴에서 **Create new file**을 선택합니다.

---

## 3단계: 폴더 이름과 '슬래시(/)' 입력하기 (핵심!)

가장 중요한 단계입니다. 파일 이름을 쓰는 칸에 다음 과정을 따라 하세요.

1. **폴더 이름 입력:** 빈칸에 원하는 폴더 이름을 씁니다. (예: `chirpy-image-path-guide`)
2. **슬래시(/) 입력:** 이름 바로 뒤에 키보드의 `/`를 누릅니다.
3. **결과:** 슬래시를 누르는 순간, 방금 쓴 이름이 폴더 형태(`📁`)로 변하면서 자동으로 다음 입력 칸으로 넘어갑니다!

---

## 4단계: 임시 파일 생성 및 저장

GitHub은 내용이 없는 빈 폴더는 저장해주지 않습니다. 따라서 폴더 안에 아무 파일이나 하나 만들어 두어야 폴더가 사라지지 않습니다.

1. 파일명 자리에 `.gitkeep`이라고 입력합니다. (내용은 비워둬도 됩니다.)
2. 화면 하단의 **Commit changes** 초록색 버튼을 클릭하여 저장합니다.

---

## 💡 요약: 경로 입력창에 쓰는 법

입력창 하나에 아래와 같이 타이핑한다고 생각하시면 쉽습니다.

> `폴더명` + `/` + `.gitkeep`

이제 `posts` 폴더 안에 여러분이 만든 새 폴더가 생겼을 것입니다. 앞으로 이 폴더 안에 해당 포스팅에서 사용할 이미지들을 차곡차곡 업로드(Upload files)하여 관리하시면 됩니다. 

글 하나당 폴더 하나! 이 규칙만 지키면 나중에 수백 개의 이미지가 생겨도 관리가 아주 편해집니다. 🐜✨

## 🛠️ 적용 방법 다시 확인하기(check list)

0. **GitHub 접속**: _posts 폴더로 갑니다.
1. **새 파일 생성**: Create new file을 누릅니다.
2. **파일명 입력**: 2026-04-07-how-to-create-image-folders-on-github.md 라고 씁니다.
3. **내용 붙여넣기**: 위 박스 안의 마크다운 내용을 그대로 붙여넣습니다.
4. **저장**: 하단 Commit changes를 누르면 포스팅 완료!
어렵게 느껴졌던 폴더 생성도 슬래시(/) 하나면 해결됩니다. 오늘도 **멋진 기록** 이어가시길 바랍니다! 🚀🐜
