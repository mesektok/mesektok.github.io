GitHub 블로그(Chirpy 테마)를 운영하면서 가장 먼저 마주치는 난관 중 하나가 바로 **이미지 업로드**입니다. 이미지는 텍스트 파일과 달리 직접 코드로 칠 수 없기 때문이죠.

오늘은 깃허브 웹사이트에서 직접 이미지를 올리는 가장 직관적인 방법을 단계별로 정리해 보겠습니다.

---

## 1단계: 업로드할 폴더로 이동하기

우선 이미지가 저장될 정확한 위치로 가야 합니다. 
GitHub 저장소(Repository)에서 다음 경로를 찾아 클릭하세요.

> **경로:** `assets` / `img` / `posts` / `chirpy-image-path-guide` /

만약 해당 폴더가 없다면, 상위 폴더에서 새 폴더를 미리 만들어두어야 합니다.

---

## 2단계: 'Add file' 버튼 활용하기

폴더 안으로 접속했다면 화면 오른쪽 상단을 확인하세요.

1. **Add file** 버튼을 클릭합니다.
2. 드롭다운 메뉴에서 **Upload files**를 선택합니다.

---

## 3단계: 이미지 파일 등록하기

이제 파일을 전송할 준비가 되었습니다.

- 중앙의 넓은 영역에 내 컴퓨터에 있는 `cover.jpg` 파일을 **마우스로 끌어다 놓으세요(Drag & Drop).**
- 또는 `choose your files` 링크를 눌러 직접 파일을 선택해도 됩니다.

---

## 4단계: 변경 사항 저장(Commit)하기

파일이 목록에 나타났다면 마지막으로 서버에 반영해야 합니다.

1. 화면 하단으로 스크롤을 내립니다.
2. **Commit changes**라고 적힌 초록색 버튼을 클릭합니다.
3. 잠시 기다리면 서버에 실제 이미지 파일이 저장됩니다.

---

## 💡 마지막 체크리스트 (매우 중요!)

이미지를 올린 후 블로그에 나타나지 않는다면 다음 두 가지를 꼭 확인하세요.

### 1. 경로 및 대소문자 일치
마크다운 파일(`_posts` 폴더 안의 글)에 적은 경로와 실제 업로드한 경로가 **대소문자까지** 똑같아야 합니다.
- **폴더명:** `chirpy-image-path-guide`
- **파일명:** `cover.jpg` (Cover.jpg와 다릅니다!)

### 2. 배포 시간 대기
GitHub Actions가 사이트를 다시 빌드하는 데 약 1~2분이 소요됩니다. 상단의 **Actions** 탭에서 초록색 체크(`v`)가 뜰 때까지 기다린 후 새로고침해 보세요.

이제 여러분의 블로그에도 멋진 썸네일과 본문 이미지가 잘 나타날 거예요! 🐜👍
